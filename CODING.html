Coding

Ce que j'apprends — notes et références personnelles

Structure de base
Squelette HTML
La structure minimale de toute page web. À coller en tout début de fichier .html.
<!DOCTYPE html> <html lang="fr"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <link rel="stylesheet" href="style.css"> <title>Titre de la page</title> </head> <body> <!-- Ton contenu ici --> </body> </html>
⚠️ Ne jamais mettre style.css deux fois dans le head — une seule fois suffit !
Commentaires HTML
Les commentaires ne s'affichent pas dans le navigateur. Utiles pour s'organiser.
<!-- Ceci est un commentaire --> <!-- ====== DÉBUT DU MENU ====== --> <nav>...</nav> <!-- ====== FIN DU MENU ====== -->
Balises courantes
Titres et paragraphes
Les titres vont de h1 (plus grand) à h6 (plus petit). Un seul h1 par page.
<h1>Titre principal</h1> <h2>Sous-titre</h2> <h3>Sous-sous-titre</h3> <p>Un paragraphe de texte.</p>
Liens et images
Les liens (a) pointent vers une URL ou un fichier. Les images (img) n'ont pas de balise fermante.
<!-- Lien --> <a href="TISANES.html">Voir les tisanes</a> <!-- Image --> <img src="Minljiva.png" alt="Logo Minljiva">
Le alt décrit l'image pour l'accessibilité — toujours le remplir.
Div et organisation
Les div sont des boîtes invisibles qui servent à grouper et organiser le contenu.
<div class="box"> <div class="card"> <p>Contenu de la carte</p> </div> </div>
💡 La class relie la balise à un style CSS. Plusieurs éléments peuvent partager la même class.
Bases du CSS
Comment ça marche
CSS = sélecteur + propriétés entre accolades. Le sélecteur pointe qui on style, les propriétés disent comment.
/* sélecteur { propriété: valeur; } */ h1 { font-size: 22px; color: #1a1a18; } .card { /* classe = point devant */ background: white; border-radius: 12px; }
Variables CSS (custom properties)
On définit des variables dans :root et on les réutilise partout. Changer une variable = changer tout le site.
:root { --vert: #1D9E75; --fond: #f9f8f5; } .nav { background: var(--fond); color: var(--vert); }
💡 C'est exactement comme ça que fonctionne le style.css de ce projet — modifier les 3 couleurs en haut change tout !
Box model — l'espace autour d'un élément
Chaque élément HTML est une boîte. Padding = espace intérieur. Margin = espace extérieur. Border = le bord.
.box { padding: 1rem; /* espace intérieur */ margin: 1rem 0; /* espace extérieur (haut/bas) */ border: 1px solid #ccc;/* bordure */ border-radius: 12px; /* coins arrondis */ }
Flexbox — aligner des éléments
Flexbox permet d'aligner facilement des éléments côte à côte ou de les centrer.
.nav { display: flex; gap: 12px; /* espace entre éléments */ align-items: center; /* centrer verticalement */ flex-wrap: wrap; /* passer à la ligne si besoin */ }
Grid — créer une grille
CSS Grid crée des mises en page en colonnes automatiquement.
.grid { display: grid; /* colonnes auto, min 300px chacune */ grid-template-columns: repeat(auto-fit, minmax(300px, 1fr)); gap: 12px; }
💡 C'est ce qui fait que les cartes HERBORISTE s'adaptent automatiquement à la taille de l'écran.
Bases du JavaScript
Variables
Les variables stockent des informations. const = valeur fixe. let = valeur qui peut changer.
const nom = "Minljiva"; // texte (string) const age = 3; // nombre (number) let actif = true; // vrai/faux (boolean) let compteur = 0; compteur = compteur + 1; // on peut modifier avec let
Fonctions
Une fonction est un bloc de code réutilisable. On la définit une fois, on l'appelle autant qu'on veut.
// Définir une fonction function show(id, btn) { // cacher tous les onglets document.getElementById(id).style.display = 'block'; btn.classList.add('active'); } // L'appeler dans le HTML : // onclick="show('html', this)"
💡 C'est exactement la fonction utilisée pour les onglets sur cette page !
Manipuler le DOM
DOM = la page HTML vue par JavaScript. On peut cibler des éléments et les modifier.
// Trouver un élément par son id const el = document.getElementById('html'); // Modifier son style el.style.display = 'none'; // cacher el.style.display = 'block'; // montrer // Ajouter / enlever une classe CSS el.classList.add('active'); el.classList.remove('active'); // Trouver tous les éléments d'une classe document.querySelectorAll('.tab');
Tableaux (arrays)
Un tableau stocke plusieurs valeurs. On peut les parcourir avec forEach ou les filtrer avec filter.
const plantes = ['Menthe', 'Thym', 'Lavande']; // Parcourir plantes.forEach(p => { console.log(p); }); // Filtrer const med = plantes.filter(p => p === 'Lavande'); // Transformer en HTML const html = plantes.map(p => `<li>${p}</li>`).join('');
💡 C'est exactement comme ça que les cartes HERBORISTE sont générées dynamiquement depuis un tableau plants.
Ce que j'ai appris
Erreurs courantes à éviter
Les petits pièges qu'on rencontre souvent quand on débute.
!
Ne pas lier style.css deux fois — une seule ligne <link rel="stylesheet"> dans le head suffit.
!
Majuscules dans les noms de fichiers — coding.html ≠ CODING.html sur certains serveurs. Rester cohérent.
!
Oublier de fermer une balise — chaque <div> doit avoir son </div>. BBEdit colore les balises orphelines.
!
Styles inline vs CSS — éviter style="..." dans le HTML. Mieux vaut tout mettre dans style.css pour garder le contrôle.

Comment penser quand on crée un composant pour ce projet.

Le titre */ .recipe-title { font-size: 15px; font-weight: 500; } /* 4. Le sous-titre */ .recipe-sub { font-size: 12px; color: var(--texte-doux); }

C'est quoi GitHub ?
Les concepts de base
GitHub est un endroit en ligne où on stocke et sauvegarde son code. Voici les mots importants à connaître.
📁
Repository (repo) — Le dossier de ton projet en ligne. Tout ton site Minljiva est dans un repo.
🌿
Branch (branche) — Une version de ton projet. Tu travailles sur la branche main — c'est la principale.
💾
Commit — Une sauvegarde avec un message qui explique ce qu'on a changé. Comme un point de restauration.
⬆️
Push — Envoyer ses commits locaux vers GitHub en ligne.
⬇️
Pull — Récupérer les derniers changements depuis GitHub vers son ordinateur.
🍴
Fork — Copier le projet de quelqu'un d'autre pour le modifier de son côté.
Mettre à jour son site — la routine
Ce que tu fais à chaque modification
Tu modifies un fichier dans BBEdit → tu veux que le site en ligne se mette à jour. Voici les étapes.
1
Modifier le fichier dans BBEdit et sauvegarder (⌘ + S).
2
Aller sur github.com → ton repo → cliquer sur le fichier à mettre à jour.
3
Cliquer sur le crayon ✏️ en haut à droite du fichier pour l'éditer directement en ligne. Ou glisser-déposer le fichier modifié dans l'onglet du repo.
4
Écrire un message de commit en bas (ex : "Ajout recette cookies") → cliquer Commit changes.
5
Attendre 1 à 3 minutes → ton site sur github.io est mis à jour automatiquement ✅
💡 Tu peux vérifier que le déploiement est terminé dans l'onglet Actions de ton repo — cercle vert = en ligne, cercle jaune = encore en cours.
Uploader plusieurs fichiers d'un coup
Quand tu as modifié plusieurs pages en même temps, inutile de les faire une par une.
1
Dans ton repo GitHub, cliquer sur Add file → Upload files.
2
Glisser-déposer tous les fichiers modifiés en même temps.
3
Écrire un message de commit (ex : "Mise à jour pages recettes et coding") → Commit changes.
Tous les fichiers uploadés remplacent automatiquement les anciens — pas besoin de supprimer avant.
Structure de ton repo Minljiva
Ce que tu dois avoir dans ton repo
Tous ces fichiers doivent être à la racine du repo (pas dans un sous-dossier).
# Fichiers obligatoires index.html # ⚠️ minuscule — c'est la page d'accueil style.css # la feuille de style commune Minljiva.png # le logo # Les pages HERBORISTE.html TISANES.html RECETTE.html AVENIR.html CODING.html # Le dossier police fonts/ Halimun.ttf
⚠️ Si index.html est en majuscules (INDEX.html), GitHub Pages affiche une erreur 404. Toujours en minuscules !
Activer GitHub Pages
La configuration (à faire une seule fois)
1
Dans ton repo → onglet Settings (en haut).
2
Dans le menu gauche → Pages.
3
Sous Branch → choisir main et le dossier / (root) → cliquer Save.
4
Ton URL apparaît : https://ton-pseudo.github.io/nom-du-repo/
💡 Cette configuration ne se fait qu'une seule fois. Ensuite chaque commit met le site à jour automatiquement.
Erreurs fréquentes
Les problèmes courants et leurs solutions
!
Erreur 404 — Le fichier index.html est en majuscules, ou GitHub Pages n'est pas activé, ou le déploiement n'est pas encore terminé.
!
Le style ne s'affiche pas — style.css n'est pas dans le repo, ou le chemin dans le HTML est mauvais. Vérifier que c'est bien href="style.css" et non un chemin absolu.
!
Le logo ne s'affiche pas — Minljiva.png absent du repo. L'uploader comme les autres fichiers.
!
Les modifications n'apparaissent pas — Vider le cache du navigateur (⌘ + Shift + R sur Mac) ou attendre encore quelques minutes.
!
La police Halimun ne s'affiche pas — Le dossier fonts/ avec Halimun.ttf doit être uploadé dans le repo, pas seulement sur ton ordinateur.
